Mission Control

Mission Control is a dynamic multi-agent workflow visualizer built on the Claude API.

It demonstrates how a single orchestrator (Opus 4.8) plans a task, writes a JS orchestration script, fans out parallel subagents, runs adversarial verifiers, and synthesizes a final report — all in real time.

Every agent tile, verifier card, and escalation is a live representation of the underlying architecture — not animation.

Orchestratorclaude-opus-4-8
Scan agentsclaude-haiku-4-5
Verifiersclaude-sonnet-4-6
Escalationsclaude-opus-4-8
1
Plan — Opus 4.8 reads your prompt and writes a JS orchestration script. The script defines which agents run, with what models and context.
2
Scan — The JS runtime executes Promise.allSettled(), launching all scan agents in parallel. Each gets a minimal context window containing only its relevant slice.
3
Verify — One adversarial verifier per scan agent challenges the findings. Contested findings escalate to pass 2, spawning a new verifier card with a stronger model.
4
Report — The synthesis agent aggregates only verified findings, ranks by severity, and produces the final structured report.

Intermediate results live in JS variables (Node.js heap) — not in any model's context window. This is what makes parallel execution cost-efficient.

Mission Control is scenario-agnostic. Swap workflow-config.js to change the workflow entirely.

Mission Control is an open portfolio project exploring dynamic multi-agent architectures with Claude.
